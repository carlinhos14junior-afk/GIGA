# 🚀 GIGANET Fibra Óptica - Guia de Deploy e Configuração

Este é o código-fonte de um site profissional, dinâmico e moderno para o provedor de internet **GIGANET**, construído com **React**, **Vite**, **TypeScript** e **Tailwind CSS v4**, preparado para integração com o bando de dados gratuito **Supabase** e hospedagem na **Vercel**.

## 🛠️ Passo 1: Configuração do Banco de Dados no Supabase

1. Crie uma conta gratuita em [Supabase.com](https://supabase.com/).
2. Inicie um novo projeto chamado **Giganet** e aguarde a inicialização da base de dados PostgreSQL.
3. No painel de controle esquerdo do Supabase, acesse a aba **SQL Editor** (ícone de terminal de SQL `>_`).
4. Clique em **"New query"** e cole o seguinte script para criar as tabelas necessárias e alimentar as configurações do site automaticamente:

```sql
-- 1. CRIAR TABELA DE CONFIGURAÇÕES DO SITE
CREATE TABLE IF NOT EXISTS site_config (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  nome_empresa VARCHAR(255) DEFAULT 'GIGANET',
  logo_url TEXT,
  whatsapp VARCHAR(50) DEFAULT '5511999999999',
  telefone VARCHAR(50) DEFAULT '(11) 4004-9999',
  email VARCHAR(255) DEFAULT 'contato@giganetfibra.com.br',
  endereco TEXT DEFAULT 'Av. Paulista, 1000 - Bela Vista - SP',
  instagram VARCHAR(100) DEFAULT 'giganet_fibra',
  facebook VARCHAR(100) DEFAULT 'giganetfibra'
);

-- INSERIR DADOS INICIAIS DE CONFIGURAÇÃO (OPCIONAL)
INSERT INTO site_config (nome_empresa, whatsapp, telefone, email, endereco, instagram, facebook)
VALUES ('GIGANET', '5511999999999', '(11) 4004-9999', 'contato@giganetfibra.com.br', 'Av. Paulista, 1000 - Bela Vista - SP', 'giganet_fibra', 'giganetfibra')
ON CONFLICT DO NOTHING;

-- 2. CRIAR TABELA DE PLANOS
CREATE TABLE IF NOT EXISTS planos (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  nome VARCHAR(255) NOT NULL,
  velocidade VARCHAR(100) NOT NULL,
  preco NUMERIC(10,2) NOT NULL,
  beneficios JSONB DEFAULT '[]'::jsonb,
  destaque BOOLEAN DEFAULT false,
  ativo BOOLEAN DEFAULT true
);

-- INSERIR PLANOS PADRÃO GIGANET

VALUES 
('Plano Smart 500 Mega', '500 MEGA', 99.90, '["100% Fibra Óptica", "Wi-Fi incluso", "Download ilimitado", "Suporte técnico humanizado"]'::jsonb, false, true),
('Plano Ultra 800 Mega', '800 MEGA', 129.90, '["100% Fibra Óptica", "Wi-Fi Dual Band incluso", "Download e Upload de alta performance", "Ideal para jogos e streaming", "Instalação expressa"]'::jsonb, true, true)
ON CONFLICT DO NOTHING;

-- 3. CRIAR TABELA DE LEADS DE COBERTURA
CREATE TABLE IF NOT EXISTS leads_cobertura (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,

);
```

5. Clique em **Run** no topo direito para executar as queries. Suas tabelas estarão prontas!

## 💻 Passo 2: Subir o Projeto para o GitHub

1. Baixe o código fonte do projeto (exporte em ZIP ou conecte ao Github pelo menu Settings da AI Studio).
2. Na sua máquina local, abra o terminal no diretório do projeto e inicialize o Git:
```bash
git init
```
3. Adicione todos os arquivos e faça o commit inicial:
```bash
git add .
git commit -m "feat: site giganet completo com gerenciador e leads"
```
4. Crie um repositório vazio no seu **GitHub** (ex: `giganet-fibra`).
5. Vincule o repositório local ao GitHub e faça o push:
```bash
git branch -M main
git remote add origin https://github.com/SEU_USUARIO/giganet-fibra.git
git push -u origin main
```

## ⚡ Passo 3: Deploy Gratuito na Vercel

1. Acesse **[Vercel.com](https://vercel.com/)** e faça login com sua conta do GitHub.
2. Clique em **"Add New"** > **"Project"**.
3. Importe o repositório `giganet-fibra` que você acabou de criar no GitHub.
4. Na tela de configuração do projeto na Vercel, clique e expanda a aba **Environment Variables** (Variáveis de Ambiente).
5. Escreva e adicione as seguintes variáveis com as credenciais obtidas nas configurações do seu projeto Supabase (em **Project Settings** > **API**):

   * **Chave 1**: `VITE_SUPABASE_URL`  
     * *Valor*: *(Insira a URL de API do projeto no Supabase, ex: `https://xxxx.supabase.co`)*
   
   * **Chave 2**: `VITE_SUPABASE_ANON_KEY`  
     * *Valor*: *(Insira a chave pública de acesso anônimo do Supabase)*

6. Clique no botão **"Deploy"**! A Vercel compilará as dependências e disponibilizará o site ao vivo em um link público HTTPS em menos de 1 minuto.

## 🗝️ Acesso ao Painel Administrativo (/admin)

Para acessar e testar o painel administrativo:
1. Deslize até o rodapé ou acesse o link do seu site e clique no botão **Área do Cliente** ou clique no link **Área Administrativa** no topo.
2. Se você já configurou as variáveis do Supabase, você poderá usar os recursos do Supabase Auth para autenticar.
3. Se você estiver testando na prévia do AI Studio (sem variáveis configuradas), o sistema de **Simulação Inteligente** salvará tudo no seu `localStorage` local automaticamente sem quebrar. É possível fazer login usando as seguintes credenciais de teste:
